What We Learned
Conclusions

Dedicated context wins. Giving each competitor its own context window scored 25/25 vs sequential's 23/25 — with 5/5 factual consistency vs 2-3/5.

Synthesis is the product. The LLM synthesis step generates the executive summary, comparative tables, and strategic implications. Cutting it cratered the score from 25 to 16.

4-5 grounded searches is the quality floor. Reducing to 2 calls per competitor dropped quality from 25/25 to 19/25.

Agent vs Script

Scripts can match agent quality. Raw parallel achieved 25/25 at lower latency (102s vs ~6min) with comparable sources.

Agents add conversational value. The ADK architecture supports follow-up questions, deeper dives, and iterative refinement — things a fire-and-forget script can't do.

Latency is the frontier. The remaining challenge: match 25/25 quality at closer to sequential's 65s speed.

How We Measured Quality

A three-layer evaluation framework: promptfoo for prompt regression testing, LLM-as-judge for head-to-head comparisons, and structured metrics for every run.

Evaluation Framework

Every architecture was evaluated on the same five criteria, scored 1-5 by Gemini acting as an impartial judge. Automated metrics tracked sources, dimensions, and structure alongside the qualitative scores.

Prompt Engineering

The prompt evolved from an underspecified instruction to a structured template with explicit dimensions, null-case handling, and output format requirements.

Before
Compare West Elm and CB2.
Be extremely detailed.

No structure. No source requirements. No null-case handling. "Detailed" is subjective. Outputs varied wildly between runs.

After
You are a competitive intelligence analyst...
Compare across 4 explicit dimensions
If NO AI features found, explicitly state
Output as: Exec Summary | Analysis | Strategic Implications | Sources

Role assignment, explicit dimensions, null-case instruction, required sources, structured output format. Consistent across runs and competitor pairs.

Design Decision Rationale
Role assignment Sets context; signals comparative analysis purpose, not generic Q&A
4 explicit dimensions Ensures consistent coverage across competitors; makes outputs directly comparable
Null-case handling Forces explicit reporting of absence vs. hallucination ("No AI features identified")
Source requirement Every claim must be verifiable; prevents confabulation
Structured output Executive summary table fits on one slide; enables automated metric extraction

Validation: Confirming Dead Ends

To be confident in our choice, we also tested two alternative approaches and confirmed they were strictly worse.

Dead End: ADK Native google_search

Tried using ADK's built-in google_search tool instead of our custom one. Scored 21/25 — close, but source quality dropped to 2/5. URLs were unverifiable redirect links, and "CB2" queries returned cannabinoid receptor results instead of the furniture brand.

Dead End: Programmatic Synthesis (no LLM)

Removed the synthesizer LLM entirely and merged research with Python templates. Sources jumped to 95, but score plummeted to 16/25. Strategic insight: 0/5. No executive summary, no comparative tables, no "so what." The synthesis LLM is the product, not overhead.

Eval Methodology Learnings
What Worked

Three-layer eval stack. promptfoo catches structural regressions at the prompt level. LLM-as-judge catches quality differences at the architecture level. Automated metrics (source count, dimension coverage) catch obvious failures instantly.

Hill-climbing workflow. Add test case for failure mode, run eval, modify prompt, re-run, confirm no regressions. Each change validated against 3 competitor pairs before shipping.

Pitfalls We Found

Judge truncation bias. Our first programmatic eval scored 11/25 — but we'd truncated the 70K-char output to 15K, which only showed one competitor. Fair truncation scored 16/25. Eval design is as important as prompt design.

Source count ≠ source quality. Programmatic had the most sources (95) but worst score. The metric that matters is whether sources are verifiable and cited in context, not how many there are.
